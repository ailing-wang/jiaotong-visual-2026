# 基于深度学习的交大地标图像检索与文字检测系统

## 一、引言

### 1.1 研究背景与问题

随着数字化校园建设的推进，大量的校园图片和视频资料涌现。如何高效地从海量图像库中检索出用户所需的特定场景或地标图片，成为智慧校园建设中的一个重要课题。同时，在校园场景中，图片中往往包含大量文字信息（如路牌、指示牌、建筑名称等），对这些文字进行自动检测与定位，对于图像内容理解、场景标注等下游任务具有重要意义。

本实验基于"交大视觉印象数据集"，分别实现**图像检索**与**文字检测**两个核心任务，旨在探索深度学习技术在校园场景图像分析中的应用。

### 1.2 研究目标

- **图像检索任务**：给定一张查询图片，从包含7728张图片的底库中检索出最相似的Top-K结果，评价指标为P@K（K=20, 40, 60）
- **文字检测任务**：对图片中的文字区域进行定位检测，采用可视化评价与人工核验相结合的方式

### 1.3 基本思路

针对上述两个任务，本实验采用以下技术路线：

1. **图像检索**：利用预训练的深度卷积神经网络（ResNet50）提取图像的全局特征表示，通过余弦相似度计算实现以图搜图功能
2. **文字检测**：采用EAST（Efficient Accurate Scene Text Detector）深度学习模型进行端到端的文字检测，并针对检测失败的情况设计了MSER（Maximally Stable Extremal Regions）备选方案

## 二、实验方案设计

### 2.1 问题形式化描述

#### 2.1.1 图像检索问题

设图像库 $\mathcal{D} = \{I_1, I_2, ..., I_N\}$，其中 $N = 7728$。对于查询图像 $Q$，图像检索的目标是找到一个排序结果 $\mathcal{R} = (I_{r_1}, I_{r_2}, ..., I_{r_K})$，使得：

$$similarity(Q, I_{r_i}) \geq similarity(Q, I_{r_j}), \forall i < j$$

性能评价指标定义为：

$$P@K = \frac{1}{|\mathcal{Q}|} \sum_{q \in \mathcal{Q}} \frac{|\{I \in TopK(q) : label(I) = label(q)\}|}{K}$$

其中 $\mathcal{Q}$ 为查询集，$TopK(q)$ 为查询 $q$ 的前K个检索结果。

#### 2.1.2 文字检测问题

对于输入图像 $I$，文字检测的目标是输出文本区域集合：

$$\mathcal{B} = \{(x_1, y_1, x_2, y_2, c)_1, ..., (x_1, y_1, x_2, y_2, c)_M\}$$

其中 $(x_1, y_1, x_2, y_2)$ 为检测框坐标，$c$ 为置信度分数。

### 2.3 模块详细设计

#### 2.3.1 图像检索模块

**特征提取器（Feature Extractor）**

采用ResNet50预训练模型作为特征提取 backbone，网络结构如下：

```
输入图像 (3×224×224)
    │
    ▼
ResNet50卷积层 (输出: 2048维特征向量)
    │
    ▼
L2归一化
    │
    ▼
特征向量 f ∈ ℝ²⁰⁴⁸
```

**关键技术点：**
- 使用ImageNet预训练权重，迁移学习提取通用视觉特征
- 去除最后的全连接层，仅保留卷积特征
- 对特征向量进行L2归一化，确保余弦相似度计算的稳定性

**检索引擎（Retrieval Engine）**

构建特征索引库，实现高效检索：

1. **索引构建阶段**：对底库中所有图像提取特征，构建特征矩阵 $\mathbf{F} \in \mathbb{R}^{N \times D}$
2. **检索阶段**：对查询图像提取特征 $f_q$，计算与底库特征的余弦相似度，返回Top-K结果

$$sim(f_q, f_i) = \frac{f_q \cdot f_i}{\|f_q\| \cdot \|f_i\|} = \cos(\theta)$$

#### 2.3.2 文字检测模块

**EAST检测器**

采用EAST深度学习模型，该模型基于FCN（Fully Convolutional Network）架构，能够端到端地预测文本区域的几何形状和置信度分数。

**网络结构：**
```
输入图像 (3×W×H)
    │
    ▼
特征提取器 (VGG16 backbone)
    │
    ├──► 几何特征图 (score map + geometry)
    │
    ▼
NMS后处理
    │
    ▼
检测结果框
```

**备选方案（MSER检测器）**

当EAST模型不可用时，采用OpenCV的MSER（最大稳定极值区域）算法作为备选。MSER通过检测图像中稳定的连通区域来定位文字，具有实现简单、运行速度快的优点。

#### 2.3.3 结果可视化模块

- **P@K曲线图**：为每个地标生成精度-召回曲线
- **检索-检测配对可视化**：展示查询图像、检索结果及其对应的文字检测结果

#### 3.2.3 文字检测结果

文字检测采用MSER检测器（EAST模型因网络问题未下载成功）：
- **成功案例**：清晰文字、规则排版的图片检测效果良好
- **失败案例**：复杂背景、低对比度、弯曲文字等场景存在挑战

检测结果可视化示例：
- 每个地标选取2组"检索结果-文字检测"配对样例
- 合计输出24组可视化结果

### 3.3 结果解释

#### 3.3.1 检索性能影响因素

1. **特征表达能力**：ResNet50在ImageNet上预训练，能提取较为鲁棒的视觉特征
2. **地标特异性**：不同地标具有不同程度的视觉独特性，如图书馆（tsg）、南门（nm）、书架（sjz）等有独特的视觉特征，检索效果较好
3. **样本数量影响**：kx（可信软件）和yk（游泳馆）仅有2张查询图片，统计意义有限，且精度较低可能与样本代表性有关
4. **yf（逸夫楼）性能较低**：可能是因为逸夫楼外观与其他建筑相似度较高，特征区分度不够

#### 3.3.2 文字检测性能分析

1. **MSER优势**：实现简单、运行速度快，对简单场景效果较好
2. **MSER局限性**：
   - 对复杂背景敏感
   - 难以检测低对比度文字
   - 对文字大小变化适应能力有限
3. **改进方向**：若能成功下载EAST模型，检测精度有望提升

## 四、总结与展望

### 4.1 工作总结

本实验完成了以下工作：

1. **图像检索系统**：基于ResNet50深度特征提取，实现了以图搜图功能，平均P@20达到0.784
2. **文字检测系统**：采用MSER检测器实现了自然场景文字检测
3. **结果可视化**：为12个地标生成了P@K曲线图和24组检索-检测配对可视化结果
4. **模块化设计**：代码结构清晰，模块解耦，便于后续扩展和维护

### 4.2 创新点

1. **双轨检测方案**：设计了EAST+MSER的文字检测备选机制，提高系统鲁棒性
2. **精细化评价**：按地标类别分别统计P@K指标，提供细粒度的性能分析

### 4.3 实验结果分析

从实验结果可以看出：
- **高精度地标**：sjz（书架）、nm（南门）、fhy（分号楼）等地标检索精度较高，说明这些地标的视觉特征具有较好的独特性和区分度
- **低精度地标**：kx（可信软件）、yf（逸夫楼）等检索精度较低，可能与样本数量少或视觉特征相似有关
- **总体趋势**：随着K值增大，P@K值普遍下降，符合预期规律

### 4.4 有待改进之处

1. **特征优化**：可尝试更先进的视觉特征（如CLIP、DINO等自监督模型）提升检索精度
2. **检测模型增强**：可引入文字识别（OCR）后处理，或采用Transformer-based检测器（如DBNet）
3. **端到端联合学习**：探索图像检索与文字检测联合优化的可能性
4. **大规模部署**：优化推理速度，支持实时检索和检测

### 4.5 下一步工作方向

- 收集更多训练数据，微调专用检索/检测模型
- 引入文本语义信息，实现多模态检索
- 优化系统性能，支持移动端实时运行
